# Code map

This map records the current ownership boundaries. Update it when target ownership or feature placement changes.

## Project-level files

- `README.md`: project intent, asset boundary, current build status, and the root one-click launcher entry point.
- `Launch FearMore.cmd`: minimal Windows entry point for the default `Modern` D3D11-wrapper launcher preset; it forwards options to `tools/runtime/Start-FearMore.ps1`, retains explicit `-Preset Stable` rollback, and owns no staging or compatibility logic.
- `Build FearMore Project Installer.cmd`: canonical minimal public-project entry point; it forwards options to `tools/public/Build-FearMorePublicProject.ps1` and owns no build or packaging logic. The former household-named command remains a forwarding compatibility alias.
- `source-patches`, `source-overlay`, and `source-scaffold`: public module-source reconstruction inputs. They contain the minimal official-SDK-relative delta, new FearMore source files, and project-owned CMake scaffold; generated `FEAR/Dev/Source` remains ignored.
- `tools/public`: public build bootstrap only. `Initialize-FearMoreModuleSource.ps1` owns transactional SDK-plus-delta reconstruction, `Get-FearMorePublicDependencies.ps1` owns guarded exact-identity acquisition under ignored `vendor-local`, `Build-FearMoreModules.ps1` owns module configuration/build verification, and `Build-FearMorePublicProject.ps1` composes those focused owners before delegating to the existing launcher/installer builders.
- `docs/building.md`: reproducible game-module build and local SDK setup.
- `docs/roadmap.md`: ordered milestones and behavior-preservation gates.
- `docs/modern-rendering.md`: ultrawide/Hor+ acceptance matrix, movie policy, supersampling budget, shadow plan, and external-renderer lane gates.
- `docs/source-provenance.md`: inherited-source identity and redistribution risks.
- `docs/reference-implementations.md`: external design references, licenses, and copy/use boundaries.
- `docs/asset-sourcing.md`: license, provenance, realism, and technical gates for external tools and project-owned content.
- `docs/free-gore-assets.md`: reviewed free gore-asset candidates, exact licenses, provenance links, and use/rejection decisions.
- `docs/enhanced-gore.md`: current source vertical slice, runtime controls, preservation guarantees, save boundary, acceptance pass, and deliberately deferred content.
- `docs/ai-timing.md`: high-framerate scheduler root cause, rollback control, preserved GOAP ownership, related sensor fix, and encounter acceptance matrix.
- `docs/controller-support.md`: source-owned SDL3 runtime boundary, default mapping, in-game options, legacy-controller coexistence, verification, and live acceptance matrix.
- `external/EchoPatch`: pinned stock-retail runtime patch and implementation reference; not part of the F.E.A.R. game-module targets and not presumed binary-compatible with rebuilt modules.
- `patches/echopatch` and `tools/echopatch`: project-owned, exact-commit EchoPatch compatibility and RTX camera-diagnostic patches, conservative engine-only profiles, bounded D3D9 telemetry overlay, and isolated GPL-side build/proof workflow. `EchoPatchPromotion.psm1` is the focused guarded mutation owner for per-output locking, durable intent/commit records, startup rollback or committed cleanup, and coherent package promotion. An unreadable intent plus any unidentified sibling candidate fails closed with all recovery evidence retained. Generated source and binaries stay ignored under `vendor-local`; explicit Rebuilt modes consume only pinned validated local packages and never the full stock profile.
- `tools/runtime`: disposable stock-EchoPatch, rebuilt-retail, optional dgVoodoo2 D3D11, isolated RTX Remix diagnostics, engine-only-EchoPatch, optional local HD-texture mounting, and SDK-layout stage creation plus separate dependency-free verification suites. `Start-FearMore.ps1` is the focused one-click preset/profile orchestrator; it delegates disposable-stage preparation to `New-FearRuntimeStage.ps1`, derives its owned `+EnhancedGore 0/1` pair from the strict saved setting, consumes the saved corpse-persistence selection without adding an EchoPatch hook, and delegates both parked RTX presets' retail launch to the guarded sidecar/Steam primitives. `FearLauncherProfile.psm1` owns explicit/primary-display seed selection and the ordered, missing-files-only `Game.ini`/`settings.cfg` transaction that prevents first-run auto-detect from replacing a new ultrawide profile; its Modern caller seeds Enhanced Gore, bounded Corpse persistence, and SDL controller input On only when creating a new profile, while the underlying compatibility defaults remain Off. It reuses `FearRuntimeStageSafety.psm1` for component-by-component and exact file-target guards. `Test-FearLauncherProfile.ps1` owns that transaction's root-cause contract, ancestor-reparse rejection, existing-file byte preservation, partial-profile behavior, and Stable/Modern mapping coverage. `Test-EnhancedGoreSource.ps1` is the non-launching source-invariant owner for the server default-off/Modern-default-on boundary, persisted Gameplay toggle, save ordering, validated replay/de-duplication, dispatch-scoped node context, finite local-single-player control bridge, and stock gore/low-violence precedence. `Test-CorpsePersistenceSource.ps1` owns the stock-Off payload, 4096/24/48 bounds, distinct radius/total eviction, strict multiplayer gore gate, new-profile-only seed, and disabled EchoPatch world-hook invariants. `Test-UltrawideCinematicSource.ps1` owns static authored-letterbox routing plus centered-aspect geometry checks without claiming visual acceptance. `Test-ClientFxHighFpsSource.ps1` owns the rebuilt particle-delta and batch-marker invariants. `Get-FearAiProfileSummary.ps1` is the read-only, invariant-culture parser and metric owner for source profiler CSVs. `Analyze-FearCameraCapture.ps1` owns read-only CameraLab arm/hook/schema/identity/order gates, source motion statistics, payload recoverability, numeric fixed-function view/projection correlation, and optional shader-CTAB mapping; it does not upgrade stale shader attribution into proof. `FearRuntimeExecutable.psm1` owns read-only PE identity, Steam-install detection, and EchoPatch LAA-pair attestation; `FearRendererPackage.psm1` owns pinned renderer archive/config inspection, the exact RTX Bridge troubleshooting profile, exact Custom/ReSTIR/DLSSG-off seed and live-config safety identities, and ZIP-path validation; `FearEnginePatchPackage.psm1` owns pinned derivative manifest/binary/config inspection; `FearControllerPackage.psm1` owns pinned official SDL3 archive/runtime/license validation, `Get-FearControllerRuntime.ps1` owns guarded acquisition, and `Test-FearPhysicalController.ps1` owns x86 physical-device/axis/button/rumble acceptance without changing game settings; `FearDdsIdentity.psm1` owns the reusable legacy-DDS parser and canonical DDS manifest hash; `FearTexturePackage.psm1` owns the HD package boundary; `FearLauncherSettings.psm1` owns strict in-game setting and launcher-side registration consumption; Runtime stages, local registrations, third-party archives, controller binaries, and texture assets remain ignored.
- AI timing verification: `Get-FearAiProfileSummary.ps1` owns broad source-profiler summaries; `Test-AiTimingSource.ps1` owns non-launching scheduler/order/profiler/serializer invariants; and `Get-FearAiEncounterAcceptance.ps1` owns bounded live-profile target-FPS tolerance, dynamic/fixed population, starvation, owner-count, and simulation timestamp/engine-delta consistency. `Test-FearAiEncounterAcceptance.ps1` protects the analyzer with synthetic frozen/jumped-clock profiles and makes no live-runtime claim.

## Build ownership

### Installer ownership

- `tools/installer`: locally generated Project Installer packaging. `Build-FearMoreProjectInstaller.ps1` is the read-mostly composition owner: it reports missing launcher inputs, verifies or delegates launcher assembly, then delegates installer compilation without reproducing either mutation boundary. The former household-named script remains a forwarding compatibility alias. `FearMoreInstaller.psm1` owns the transactional, manifest-verified Launcher install/removal boundary; `Install-FearMore.ps1` composes prerequisite, registration, Modern preparation, and optional guided HD bootstrap owners; `Ensure-FearMoreVCRuntime.ps1` acquires only Microsoft's signed v14 x86 redistributable when missing; `FearMore.iss` owns per-user shortcuts/uninstall and the optional ignored Stable Lite payload; and `New-FearMoreInstallerPackage.ps1` validates all local inputs before compiling a disk-spanned artifact. The installer never owns retail files, player saves, or generated runtime stages.

- `FEAR/Dev/Source/CMakeLists.txt`: top-level lane selection and isolation checks that prevent the F.E.A.R. stock-module build from co-building inherited engine/game lanes.
- `FEAR/Dev/Source/CMakePresets.json`: supported Visual Studio 2022, v141, Win32 Debug and Release entry points.
- `FEAR/Dev/Source/FEAR/CMakeLists.txt`: F.E.A.R. module ordering and validation of the local Public Tools source input.
- `FEAR/Dev/Source/cmake/FearLegacyProject.cmake`: shared legacy-target policy, configuration-correct `_DEBUG`/`_FINAL` and CRT selection, old `.vcproj` manifest parsing, SDK include precedence, output paths, and configuration-aware import-library mapping.
- `FEAR/Dev/Source/cmake/FearLegacyCompilerCompat.h`: shared compiler and retail-STL-ABI compatibility surface forced into modern-toolchain legacy translation units.
- `FEAR/Dev/Source/cmake/FearLegacyCrtCompat.cpp`: Debug-only VS2003 `_snprintf` import-pointer ABI bridge required by the official `Shared_Assert` archive.
- `FEAR/Dev/Source/FEAR/LegacyStdLith` and `LegacyPlatform`: F.E.A.R. Public Tools versions of shared libraries whose ABI differs from the older merged tree.
- Module-local `CMakeLists.txt` files under `FEAR/ClientFxDLL`, `FEAR/ClientShellDLL`, `FEAR/ObjectDLL`, `FEAR/Shared`, and `FEAR/Libs/LTGUIMgr`: target creation, definitions, includes, and dependencies. Where a target calls `fear_sources_from_vcproj`, its tracked legacy `.vcproj` remains the authoritative source-membership manifest; `ObjectDLL/Game_ServerShell.vcproj` now owns both `AIProfiler.cpp` and `AIProfiler.h`, and `ObjectDLL/CMakeLists.txt` derives them without a second manual append.

The shared CMake helper is the established build primitive. Module-specific source ownership remains beside each module rather than accumulating in the helper.

## Inherited source

### `FEAR/Dev/Source/FEAR`

- `ClientShellDLL`: retail client game module, player camera, interface/HUD, client character effects, model decals, shell casings, and client save/load.
  - `InterfaceMgr::UpdatePatchInfo` owns the obsolete GameSpy auto-patch request. Modern-toolchain builds complete that request locally because its engine interface embeds an incompatible VC7.1 `std::string`; the original project retains the historical call.
  - Gameplay, zoom, and real-time cinematic Hor+ projection ownership is shared by `InterfaceResMgr::GetScreenFOV` and `PlayerCamera`. `InterfaceResMgr::GetAspectConstrainedRect` owns reusable centered maximum-aspect geometry; `InterfaceMgr::UpdateLetterBox` applies it as an opaque final side mask for authored letterboxed cameras and for `PlayerCamera`'s narrowly identified scripted PlayerLure presentations (valid lure FX, special animation, and authored crosshair off) on outputs wider than 16:9. `SetLetterBox` remains the sole top/bottom-bar owner. Live CameraFX (including `CT_FULLSCREEN`), gameplay, and ordinary or interactive lures remain full-width.
  - `FearMoreCameraProbe` owns the opt-in, bounded, source-side main-camera JSONL seam plus edge-only console diagnostics for scripted-lure side-mask transitions. Schema 2 keeps normalized projection XY separate from camera depth; the runtime analyzer remains read-compatible with schema 1 evidence. The first/every-sixtieth/final frame batch policy also flushes once on the enabled-to-disabled edge without closing or truncating the trace on re-enable.
  - `FearMoreGraphicsSettings` owns the shared exact CVar/profile translation for Modern renderer quality, effects-target quality, post-processing, and HUD placement. `ScreenDisplay` owns the existing option-row controls and restart guidance; the launcher consumes saved restart-bound values without making the UI mutate files or runtime packages.
  - `FearMoreControllerMgr` owns dynamic SDL3 lifetime, physical-device discovery, normalized controller state, and rumble submission. `BindMgr` merges that state into the existing command bindings and suppresses duplicate legacy gamepad bindings only while SDL input is active. `ScreenJoystick` owns the in-game toggle, sensitivity, radial deadzone, invert-Y, and vibration options through the existing screen/profile primitives. `PlayerCamera` bridges authored ClientFX vibration to controller rumble without changing the effect records.
  - `FearMoreDefaultBindingResolver` owns the narrowly scoped fallback from the retail default database's English non-alphanumeric keyboard names to stable DirectInput control codes. `ProfileMgr` keeps exact localized name resolution first, enables the fallback only for new-profile/default-database loads and explicit Restore Defaults, and continues to save the actual localized object names. Ordinary existing-profile loads, custom mappings, and deliberate unbindings stay on the original exact-name path.
  - `CursorMgr` owns the engine hardware-cursor handle lifecycle across focus and renderer teardown. It retires the tracked handle on those lifecycle events, reloads the authored default cursor before use, and falls back to `CM_None` rather than calling the retail engine with a null handle; the existing hardware-cursor preference and menu-state ownership remain unchanged.
  - `FearMoreCorpsePersistence.h` owns the single client-side cvar name and bounded 4096/24/48 corpse budget. `GameClientShell::SendPerformanceSettingsToServer` preserves the original `BodyCap*` payload when Off and substitutes those constants only for local single-player when On.
  - `FearMoreRemasterQuality` owns the atomic retail performance-record compatibility preflight and Maximum-level queueing for the focused remaster preset. `ScreenPerformance` owns only its opt-in control, source help, feedback, post-queue UI refresh, and active-profile seed for its hidden resolution fields; leaving that screen retains the established apply/save boundary without reapplying stale bootstrap dimensions.
  - Pre-rendered movie fitting and aspect-preserving black bars belong in `InterfaceMgr::UpdateMovieState`.
  - Renderer-reported resolution selection and exact profile persistence belong in `ScreenDisplay` and `ProfileMgr`; do not add a hardcoded ultrawide allowlist.
  - `ScreenGame` owns the Enhanced Gore and bounded Corpse persistence selections through the existing `CBaseScreen::AddToggle` primitive, plus the restart-bound HD-texture selection through `CBaseScreen::AddCycle`. The existing free `SaveSettings()` profile function owns their persisted console settings. Both gameplay changes reach the local server on the next world load. The launcher/stage owns Enhanced Gore's launch pair and seeds Corpse persistence only for a genuinely new Modern profile; it never rewrites existing settings.
  - `CharacterFX::HandleSeverMsg` owns stock/live and persisted-replay sever presentation. It resolves a valid sever-body piece before committing location de-duplication; low-violence handling retains its existing hidden-body consumption path.
  - The centered 16:9 HUD safe-area transform is owned by `InterfaceResMgr`; `HUDItem` applies it to left/right edge alignment, and `HUDSafeAreaFullWidth` is the legacy-placement override. Raw HUD layouts and menu layout/hit testing still need migration to the same transform. Fullscreen and world-space consumers stay on explicit full-viewport paths.
  - Active source-owned render-target and volumetric-light quality belongs in `RenderTargetGroupFx`, `RenderTargetFX`, and `VolumetricLightFX`. Generic mirror/reflection groups preserve authored LOD dimensions because their material projection/sampling contract is not resolution-independent. The volumetric resource manager alone owns the optional doubled shadow target and keeps requested, authored-native, and actually allocated sizes distinct so allocation fallback and cache invalidation remain coherent. Broad retail shadow-volume/blur settings remain performance-database/closed-renderer behavior.
  - Modern world render scaling belongs in `PlayerCamera` and the shared renderer target path.
  - Shared HUD scaling belongs in `InterfaceResMgr`, `HUDItem`, `HUDMgr`, and `LayoutDB`, not individual HUD widgets.
- `ObjectDLL`: server game module, characters, damage, AI, navigation, squads, goals, actions, sensors, and authoritative sever decisions.
  - Enhanced Gore damage-zone state and corpse-damage policy belong here. `Character` owns pending model-node context, and `Projectile::ImpactDamageObject` closes the dispatch for paths that send no direct damage so a later message cannot inherit stale gore location state.
  - `Character::CapNumberOfBodies` owns radius and level-wide corpse eviction. Radius-selected bodies must be tracked and skipped by the total queue so every counted fade is distinct; permanent and already-fading bodies retain their inherited exclusions.
  - `GameServerShell` owns the local-single-player runtime-control bridge. It applies only names from `Shared/FearMoreRuntimeControls.h` and accepts only fully parsed, finite, float-range values.
  - `AIProfiler` owns opt-in, per-authoritative-server-frame CSV aggregation. `GameServerShell::PreUpdate`/`PostUpdate` own the frame boundary; existing AI, manager, sensor, goal-selection, and navigation owners contribute focused timing scopes without changing behavior.
  - Existing GOAP, goal, action, sensor, squad, and authored-navigation boundaries are protected.
- `ClientFxDLL`: reusable visual-effect runtime. Its source-owned particle update path accepts positive sub-millisecond frame deltas, and zero-expired batch markers preserve authored update time without pre-aging. Extend existing effect records/lifetimes before adding a parallel gore-effect framework.
- `Shared`: game databases, shared enums, messages, versioning, and data definitions used by client and server.
  - Stable damage-zone, hit-location, sever-piece, and message identifiers shared by the implemented Enhanced Gore path remain here.
  - `FearMoreRuntimeControls.h` owns the exact client/server allowlist for local single-player acceptance controls; transport and authorization remain in the existing connection managers and server shell.
- `Libs/LTGUIMgr`: F.E.A.R.-specific GUI implementation. Do not substitute the older root GUI library without an API comparison.

### `FEAR/Dev/Source/runtime`

- Legacy LithTech/Jupiter engine implementation.
- `render_a/src/sys/d3d`: D3D9 device, presentation, render-target, and resolve ownership.
- `render_a/src/sys/d3d/rendershadowlist.cpp`, `rendererconsolevars.h`, and `render_a/src/sys/shadows`: dormant generic-renderer projected-shadow implementation. It is useful archaeological evidence, but `BUILD_FEAR` disables this engine lane; do not treat its console variables or limits as active retail F.E.A.R. behavior without runtime proof.
- `client`, `server`, `sound`, and `ui`: engine subsystems with independent compatibility risk.
- The engine port must remain a separate build lane until compatibility with the F.E.A.R. v1.08 SDK interfaces is proven.

### `FEAR/Dev/Source/libs` and `sdk`

- Shared legacy libraries, public engine interfaces, and several inherited binary import/static libraries.
- Do not globally delete or ignore binary libraries: first identify their expected symbols and provenance.
- New third-party dependencies must be represented by explicit imported targets rather than hardcoded machine paths.

## Local-only protected areas

- `FEAR/Dev/Runtime`: runtime data, maps, models, textures, audio, binaries, and database output.
- `FEAR/Dev/Tools`: compiled editors/plugins and internal legacy scripts.
- `FEAR/Doc`: binary legacy documents with unclear redistribution status.
- `retail` and `local-runtime`: user-owned installation and staging overlays.
- `vendor-local`: official Public Tools source and other provenance-restricted local build inputs.

Do not commit files from these paths without an explicit asset/provenance review.

## Placement rules for planned work

- Build ownership: focused module-local `CMakeLists.txt` files plus the existing `cmake/FearLegacyProject.cmake` compatibility primitive.
- Modern display: existing `InterfaceResMgr`, `PlayerCamera`, `InterfaceMgr`, HUD/LayoutDB, and renderer target owners. External `d3d9.dll` translators/injectors use separate runtime lanes rather than entering these modules.
- Optional content: in-game selection stays with the relevant screen/profile owner. `FearTexturePackage.psm1` owns pinned Full/Lite identities, `New-FearHdTextureLitePackage.ps1` owns guarded local Full-plus-official-Lite derivation, and `Register-FearHdTexturePack.ps1` owns schema-2 per-mode registration. LAA/runtime selection, exact mount ownership/no-write policy, and archive precedence stay in focused `tools/runtime` modules and the sole stage orchestrator. Third-party assets never enter `Game` or Git.
- Enhanced Gore and corpse budgets: existing damage, sever, model database, character FX, model decal, attachment, physics, performance-message, and `Character::CapNumberOfBodies` owners. EchoPatch world persistence is a separate retail-only reference lane, not the rebuilt implementation.
- AI correctness: existing scheduler and AI update path. Do not introduce a second planner or navigation stack without measured evidence.
- Project-owned tests and audit tools: `tests` or `tools` at repository root, separated from inherited engine tools.
